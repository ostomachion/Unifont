Unifont Glyph Modifications and Extensions

Emoji Consolidation and Coloring

These edits aim to make the Unifont emoji glyphs both internally consistent and reasonably consistent with the familiar designs provided by other fonts and vendors. This work also extends Unifont emoji glyphs by including:

  • glyphs for the basic emoji sequences: keycap, flag, tag, and RGI modifiers
  • glyphs for the emoji RGI ZWJ sequences: family, role, gender, hair, and other
  • glyphs for the emoji variation sequences: text style and emoji style
  • colored versions of each emoji style glyph

Color Palette

The color palette used for the color glyphs, called “Human16”, was carefully designed with the following goals:

  • Be simple enough to fit with the look, feel, and philosophy of Unifont. Human16 uses only 16 colors (not counting transparency), with no two colors being too similar. An efficient glyph encoding can be derived analogous to the .hex format used for existing Unifont glyphs, using four bits per pixel instead of one. Either a simple transparency mask can be used, or a simple algorithm can determine the transparent pixels can be automatically determined as none of the outermost pixels of the glyph will ever be pure white.
  • Be compatible with all existing emoji characters and their expected presentations. Human16 includes the 12 colors of the explicitly colored shape emoji (hearts, squares, and circle), as well as the five skin tones from the set of skin tone modifier emoji (one of which overlaps with the basic “brown” color).
  • Be accessible. Human16 incorporates the colorblind safe palette used by Bang Wong (original source; see also Wong, B. Points of view: Color blindness. Nat Methods 8, 441 (2011) for more details). This palette is expanded by adding a carefully selected shade of purple and gray.
  • Be inclusive. Rather than the Fitzpatrick scale, Human16 derives its skin tone colors from Monk, Ellis. “Monk Skin Tone Scale,” 2019. This scale is more modern, inclusive, and representative than the Fitzpatrick scale which is primary a dermatological tool and has been criticized for its Eurocentric bias and insufficient representation of global skin color diversity. The specific tones have been modified to better match the the rest of the palette.

Glyphs

The following lists the new glyph designs. The Unifont column shows the current glyph in Unifont 16.0.02. The New columns show the new glyph designs in text style, monochrome emoji style, and color emoji style. Only characters with emoji variation sequences have text style variants. Note that sequences never have text style presentations (ignoring the special case keycap sequences).

If the character has a default emoji presentation, the emoji column will be outlined in yellow; otherwise, the text column will be outlined. If Unifont adopts these new designs, they will have to decide which variants to use. Listed below are a few possibilities:

  • Use the default style (outlined in yellow). This aligns with many of the glyph designs in Unifont 16.0.02. This may align the most with user expectations in most cases, although a few edge cases may cause confusion.
  • Always use the text style when one exists. This may be a good choice considering that Unifont does not have any “support” for emoji other than providing monochrome glyphs for the base characters.
  • Always use the emoji style. This brings consistency to all emoji glyphs, although the text style may be preferred or expected in some cases.
  • Incorporate both styles by adding support for emoji variation sequences to Unifont. This gives users and systems control over the glyphs at the expense of the simplicity of Unifont.
  • Create separate fonts for text style vs emoji style. There is precedence for this approach with the Japanese versions of the CJK glyphs. A similar approach could be used for emoji at the expense of multiplying the number of charts and fonts. It would also have to be decided whether the text style version of the font would include emoji code points which do not have text styles.

Regardless of if and how Unifont uses these glyphs, I intend to make two new fonts to compliment Unifont. One will use the monochrome emoji style glyphs and the other will use the color emoji style glyphs. Both will include full emoji support for sequences and variations. These fonts will only contain emoji characters. Unifont is free to use, redistribute, or even take over these fonts when they become available.
